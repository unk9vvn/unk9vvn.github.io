{% step %}

{% endstep %}

{% step %}

{% step %}

{% step %}
Identify the "Protocol Transcoding" architecture. External clients cannot speak binary HTTP/2 gRPC natively. The Gateway receives JSON over HTTP/1.1, transcodes it to protobuf, invokes the internal gRPC service, and transcodes the response back to JSON
{% endstep %}

{% step %}
Investigate the Global Error Sanitization bottleneck. To prevent internal stack traces from leaking to the public internet, the API Gateway implements a centralized error masking policy. It intercepts upstream errors, strips the debugging payloads, and returns a generic response to the user
{% endstep %}

{% step %}
Analyze the status code mapping logic. In gRPC, exceptions are returned as `google.rpc.Status` objects. An unhandled exception yields an `INTERNAL` status (which maps to HTTP 500). A validation failure yields an `INVALID_ARGUMENT` status (which maps to HTTP 400)
{% endstep %}

{% step %}
Discover the fatal error-handling optimization: The Gateway engineers assume that internal stack traces will _only_ ever be attached to unhandled `INTERNAL` (HTTP 500) errors. They explicitly configure the transcoder to sanitize HTTP 500 responses but allow `INVALID_ARGUMENT` (HTTP 400) responses to pass through completely unredacted
{% endstep %}

{% step %}
Understand the architectural assumption: The engineers assume HTTP 400 responses are exclusively generated by deliberate business-logic validation checks (e.g., `Email is required`), which are safe to return to the client. They fail to account for early-pipeline protocol binding failures
{% endstep %}

{% step %}
Formulate the Asymmetric Exfiltration payload. You must force the internal gRPC microservice to crash _before_ it reaches the business logic controller, triggering a framework-level deserialization failure
{% endstep %}

{% step %}
Identify an endpoint requiring strict data types (e.g., an Integer ID, a Boolean flag, or a strongly-typed Enum)
{% endstep %}

{% step %}
Construct a mathematically valid JSON payload, but deliberately supply a wildly incompatible data type (e.g., submitting a massive nested object or a string array to an Integer field)
{% endstep %}

{% step %}
Submit the payload to the REST API Gateway
{% endstep %}

{% step %}
The Gateway transcends the payload to gRPC. The downstream gRPC framework's protobuf deserializer attempts to map the invalid JSON structure into the strict Protobuf message
{% endstep %}

{% step %}
The deserializer crashes. Because this is a binding failure, the gRPC framework natively catches it, assigns an `INVALID_ARGUMENT` status, and (in loosely configured environments) appends the raw parsing stack trace to the `google.rpc.DebugInfo` metadata trailer to assist developers
{% endstep %}

{% step %}
The Gateway receives the response, maps `INVALID_ARGUMENT` to HTTP 400, notes the 400-level status, bypasses its HTTP 500 sanitization filter, and blindly serializes the unredacted internal stack trace into the external HTTP response
